### 3. PMB — Local-first memory for AI · ⭐⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/pmb-local-first-memory-for-ai) | [官网](https://www.producthunt.com/r/XHBXGWG3FHAQJZ)

**定位**：通过 MCP 给 Claude Code / Cursor / Codex / Zed 一层持久化的项目记忆。
决策、教训、目标、最近工作、项目事实和文档存在磁盘上的一个 SQLite workspace 里，
读路径**不走网络也不 call LLM**。开源、offline-first、可导出。

**差异点**：memory / RAG 方向已经拥挤（Mem0、Letta、Zep、Cline 自带 memory bank
等），但绝大多数把状态藏在云端或至少要跑 embedding。PMB 明确选了 SQLite + 本地
仪表盘 + 「memory 是否真的有帮助」的 impact tracking——这是把记忆当作 dev 工件
来对待，而不是当作 LLM 的黑盒扩展。

**技术判断**：SQLite 意味着 schema 定义清晰、可 diff / grep / merge，读路径无
LLM 意味着 hit rate 完全由检索策略决定（大概率是关键词/结构化字段而不是纯向量）。
这套设计对偶尔需要在 flight mode 或私有代码库上跑 agent 的团队非常合胃口。
「impact tracking」是我最看好的部分——它承认记忆系统的最大问题是分不清哪些条目
真在被使用。

**信号价值**：**几乎是 CLAUDE.md 关注领域的直接命中**——local-first、开发者工具、
MCP、可审计。是我今天最想 star 的项目。做 agent memory 方向的强烈建议对照它的
schema 和评估思路，尤其是 impact tracking 的实现细节。

**差异点**：这类「agent as first-class internet identity」的思路一直有人在提，

一层。Mailgent 打包成一个 curl 就能拉起的服务，是我第一次看到 x402 (Coinbase 的
HTTP 402 付费扩展) 被塞进 agent infra 底座的实际产品。

**技术判断**：技术上不难做——每个都是现成协议——但组合成一个「agent 出生就有
身份、能付钱、能收邮件」的最小 bundle，产品判断是对的。要担心的是 USDC 钱包的
合规和 abuse（agent 帮别人洗钱？）以及自定义 DID 生态里的信任来源。

**信号价值**：**agent infrastructure 的边界正在从「工具编排」扩展到「身份 +
支付 + 通信」**。x402 / agentic payments 值得关注，2026 下半年是不是会有更多
API 靠 402 收费是一个可测试的假说。做 agent 方向的应该至少熟悉它 packaging
的这几个原语。

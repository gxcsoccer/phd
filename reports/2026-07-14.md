### 1. PgDog · ⭐⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/pgdog) · 234v · 27c · maker: Lev Kokotov

**定位**：开源的 PostgreSQL 前置代理，一体化实现连接池、负载均衡和分片，
对上游应用完全透明——协议层伪装成 Postgres 本体，业务代码不改一行。

**差异点**：PostgreSQL 生态里连接池（PgBouncer / PgCat）、只读副本路由
（HAProxy + Patroni）、分片（Citus / Vitess-for-PG）向来是三个各自为政的
组件，通常靠一堆胶水拼在一起。PgDog 直接把三层合并成一个 protocol-aware
的 proxy，且宣称 Postgres-compliant（不像 Citus 是分叉版）。作者 Lev
Kokotov 是原 PgCat 作者，那是 Rust 版 PgBouncer 的事实标准之一，这次是
带着经验直接做下一代。

**技术判断**：从命名和作者背景推断很可能仍是 Rust。Protocol-level proxy
意味着他们要正确处理 extended query protocol、prepared statements、事务
边界和 COPY 流——这些是压死 PgBouncer transaction mode 的历史坑。能把
sharding 做进 proxy 而不破坏 SQL 兼容，需要 query parsing + rewriting
（大概率借用 pg_query），是真正的数据库工程项目，不是套壳。

**信号价值**：这是今日榜单里少有的"底层基建"级产品。Postgres 已经吃掉
OLTP 大半市场，Neon/Supabase/PlanetScale-for-Postgres 让 Postgres 变成
默认云 DB，配套的"如何在不破坏应用的前提下横向扩展"就成了下一波空缺。
如果 PgDog 兑现了 Postgres 兼容+分片的承诺，会是同类 OSS 里的稀缺品。
做 agent / dev tools 的方向上，DB proxy 这层未来大概率也会往 "AI-native"
方向走（观察、优化、自动索引建议等）——这类基础设施先落地，才有上面的想象力。

**信号价值**：agent-native economy 的关键一层。一旦 agent 能自主付钱，

**定位**：P2P + E2EE 的文件传输，无云存储、无账号、无容量限制。移动+桌面
跨平台，一个 code / QR 就直接对传。

**差异点**：这条赛道过去几年被 Snapdrop、LocalSend、Wormhole.app 分掉，
Altersend 的差异在于"P2P + 加密 + 完整跨平台"三合一，且强调不依赖
Wi-Fi 局域网（这是 LocalSend 的短板）。技术上应该是 WebRTC data channel
+ signaling server，唯一的信任面是 signaling 服务器不能被动嗅探元数据。

**技术判断**：几乎肯定是 WebRTC + Rust/Go 客户端 + 简单的 signaling
服务。E2EE 说明密钥不上传到中间层，两端直接 ECDH 换钥、AES 或
ChaCha20-Poly1305 加密——这是 Wormhole / Magic Wormhole 类协议的标准做法。
开源使得这个信任模型可验证。

**信号价值**：今日 Top 20 里唯一贴 privacy 标签的产品。云盘 / Dropbox
向 AI 助手（"AI-organize your files"）狂奔的同时，一批人反向选择做
"完全不上云"的 P2P 工具，符合 local-first 长期趋势。做本地 AI /
on-device 推理方向的人可以观察这条支线——传输、同步、协作层未来大概率
需要非云、E2EE 的原语支撑（而不是继续绑 Dropbox / iCloud API）。

### 4. NTSC-RS · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/ntsc-rs) | [官网](https://www.producthunt.com/r/RQYYBDC3IALPXU) · 160v / 9c · `open-source / tv / github / photo-video`

**定位**：用 Rust 重写的 NTSC/VHS 视频信号模拟器。把视频"做旧"成
80–90 年代电视机的模样：扫描线、色彩偏移、磁带噪声、tracking 抖动
全部参数化复刻。可作为浏览器应用、独立 app 或 DaVinci Resolve /
After Effects 插件。

**差异点**：市面上 VHS / CRT 滤镜很多（FilmConvert、CCTV，剪映特效），
但绝大多数是预设好的 LUT + 几层叠加；ntsc-rs 是真的在做信号链路的
数字孪生——把 NTSC 编解码的每一步（luma / chroma 分离、亚载波相位
误差、磁迹）显式建模。这是为什么它能做插件而其他做不了——给
专业调色师暴露的是 colorspace-correct 的参数。

**技术判断**：Rust + WebAssembly 部署，原生跑得通 DaVinci OFX 插件，
属于"该用 Rust 就用 Rust"的典型——视频信号处理 per-pixel per-frame
开销大，C/C++ 之外只有 Rust 能写得既快又安全。开源 + 跨多个剪辑
软件 SDK 适配，工作量很扎实。

**信号价值**：和 AI 无关，但是非常清晰的"工程深度优先"信号——
让我想起前阵子的 ffmpeg.wasm。在 PH 满屏 AI wrapper 的环境里，这
种"我把一个 niche domain 用现代工具链彻底重做了"的产品反而值得
关注。也提醒：novel I/O modalities 不一定是 spatial / voice，
**复古媒介在做内容的人那里仍是稀缺品**。

但端到端 quality consistency 是真功夫。"40+ 语言一次出"的提法对
亚洲语种（中文、印地、阿拉伯）的发音质量打个问号。需要看示例。

**信号价值**：本身不在 CLAUDE.md 优先线上，但反映"AI media 从效果展
示进入工业化交付"——这条赛道的胜负手已经不是 demo 多惊艳，是
单位时长成本和 QC pipeline。Vaani 选择把自己定位为 infrastructure

发**：当某个 AI 能力进入"成本是关键"的阶段，赢家是把 cost-per-unit
打下去的人，不是把质量推到最前沿的人。

### 2. QApilot's CoWork · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/qapilot) | 306v / 64c · ratio 0.209

**定位**：把现有的人写测试用例（不是 prompts），用 AI 转成可在真机
（iOS / Android / Flutter）上执行的移动端自动化脚本；当 plan 失败时
人来 approve 一次"重新规划"，再继续跑。
**差异点**：和 Mobot / Waldo / Appium 的差别有两层：(1) **输入是
已有的人类测试用例**，不是录屏或 prompt——降低 QA 团队的迁移成本；
(2) **failure → AI replan → human approve → retry** 是一个明确的
human-in-the-loop 协议，而不是端到端黑箱。
**技术判断**：跨 iOS / Android / Flutter 三种 runtime 的 grounding
本身就难（DOM、native AX tree、Flutter 的 semantic tree 各不相同），
"3x 速度"是市场承诺，但"human-approved replanning"这条产品决策本身
是有工程经验做支撑的——说明团队踩过端到端 agent 在真机上飘的坑。
**信号价值**：今年 mobile QA agent 已经出过好几个，但大多走"prompt →
脚本"，QApilot 走"已有用例 → 脚本"，承认了存量 QE team 才是付钱方。
**这是 agent 产品里少见的"以已有 artifact 为输入"的入口选择**，对
做 dev agent 的有借鉴意义：往往最值钱的入口是别人已经写好的资产，
不是新建一个 prompt。

**差异点**：当前主流方案要么走云端 platform（CrewAI、AutoGen Studio），

routing 这点也比 hard-coded role 灵活，更容易让用户自己组合工具集。
**技术判断**：local-first、文件系统持久化、modular runtime——是真
engineering，不是 wrapper。文案里强调"absolute structural control"

topic + open source 是认真的信号。
**信号价值**：CLAUDE.md 三个关注领域同时命中——多 agent 编排、
开发者工具、local-first。这周已经看过 Polygraph / Heron / Milestones

### 4. Nada · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/nada-2) | 246v / 27c · ratio 0.110

**定位**：哼唱 / 吹口哨 / 唱进麦，直接转 MIDI。在手机上接着 arrange
旋律、选乐器、做歌曲。**作者明确写了 "No AI generation involved"**。
**差异点**：和 Suno / Udio / RIFFusion 这一类"生成式音乐"反着走——
不是让模型替你写歌，而是把人的旋律捕捉成可编辑的 MIDI。和 Vocal2MIDI
/ AudioCipher 比，重点放在"移动端、creator-loop"而不是离线分析。
**技术判断**：从语音 → 音高轮廓 → MIDI 是经典 DSP（pYIN /
CREPE / pitch tracking），加上单调旋律的量化和节拍对齐。能在
手机上做实时 + 不依赖云端的话，是一个干净的工程作品。
**信号价值**：在"全自动生成"浪潮里反着走，**显式声明"没有 AI 生成"**
作为定位的产品越来越多——这本身就是一个信号。延伸到工具领域：
"AI-augmented capture（把人类意图变成结构化产物）" vs "AI-generated
output（替你做完）"的对立，可能会成为 2026 后半的产品分水岭。

**差异点**：单功能的 CRT 滤镜或主题包不新，但 RetroMac 是

注册 CMIO/V4L 风格的 system extension。两块都是要踩坑的活，能
做出来说明作者懂 macOS 系统层。开源 + 一次性付费是健康商业
模型，没有挂 AI 的羊头。
**信号价值**：纯非 AI 的 native 系统工具今年在 PH 上越来越少
（GitHub 标签下今天 36 个、几乎全是 dev-tools），RetroMac 提醒：
"懂系统 + 审美 + 一次性买断"这条窄路依然有 200v 级别的关注度。
做 dev tool 的别只盯着 AI agent，桌面工具的"基础设施 + 趣味"
路线还有空间。

### 1. Auriko · ⭐⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/auriko-2) | Trading desk for LLM calls · 558v / 78c

**定位**：把不同 LLM 提供商当成交易场所，按 token 单价、cache 命中、延迟、
可靠性和请求质量做实时 arbitrage 路由。团队声称对比直连供应商能省 30%。

**差异点**：和 OpenRouter / Portkey / LiteLLM 那套 gateway 不同——那些工具
主要卖"一套 SDK 打通多家 API"，Auriko 卖的是"我知道你的 request 长什么样
所以我会给你选最便宜的路径"。核心竞争力是校准，不是接线。创始团队是前 quant
trader，把 spread arbitrage 那一套搬进推理市场，比通用 gateway 有更明确的
优化 loss function。

**技术判断**：真活。要做他们宣称的事情，需要（a）对每家 provider 的定价、
cache 行为、限速、reliability 做持续实测；（b）根据 request 特征
（长度、system prompt 稳定性、并发模式）预测每条路径的期望成本和期望延迟；
（c）在这个 pareto front 上做在线优化。任何一环偷懒都做不出 30% 的差距。
如果他们真的每天在跑那样一套 benchmark，这是有工程壁垒的产品。

**信号价值**：LLM 推理正在被显式建模成金融市场——单价、流动性、执行质量、
库存都开始出现。做 agent 平台的方向可以思考：如果一天几百万次 LLM 调用的
路由决策是可以套利的，那么"AI infra"这个赛道的收益结构本质上是撮合费而
不是订阅费。这一层可能会被独立厂商吃掉，也可能被主要云厂并进各自的 gateway。

### 4. Aura: Agents + Git + Intent · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/aura-agents-git-intent-open-source) | OSS IDE for controlling AI coding agents with built in loops · 178v / 33c

**定位**：Git-native 的开源 IDE，专门跑 AI 编程 agent。核心功能是把 agent
的改动按函数/类粒度和"原始任务意图"对比，在提交前判断任务是不是真的完成了。

**差异点**：Cursor / Windsurf / Cline 这一波编程助手都是"聊天窗口 + diff
review"的模式，用户还是得逐行看变更然后决定提交。Aura 换了一个 review
单位：**不是行，是 intent**。它把任务意图作为一等公民追踪，然后判断 agent
的 diff 是不是达成了那个 intent。这对多 agent 长任务的 review 是关键的一层
抽象。

**技术判断**：要做"函数/类粒度追踪"意味着它有一个静态分析层，能把 diff
落到 AST 节点上；再往上跑一个 LLM 判断"这些节点的变化是否满足 intent"。
主要挑战是 intent 的表征——一句自然语言 prompt 和一段代码对不上号，需要
中间产物（子任务、验收标准）承接。看得出是给"多 agent 并行、异步开发"设计的
基础设施。

**信号价值**：coding agent 的下一步竞争不是"谁能写代码"，而是"谁能验证代码
是否正确完成任务"。这已经不是 IDE 层面的战争，是把 CI + code review + issue
tracker 缝在一起的产物。开源版本本身也是招募早期贡献者的战术信号，值得
跟踪。

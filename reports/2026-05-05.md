不是再造一个 Cursor，而是把"Cursor + 多 agent 协作 + 模型评测"整合在
一个开源插件里。

**差异点**：当前主流 IDE agent（Cursor、Windsurf、Copilot Workspace）
都是单 agent 串行，多模型切换也只是配置项。Kilo v7 把"多 agent 并行
+ 多模型横向比较"放到 UI 主路径里，对应的是开发者已经开始把 coding
agent 当工具集而非单一助理来用的趋势。基于 OpenCode（sst 出的开源
agent 内核）也意味着不再绑死自家执行栈。

**技术判断**：portable core + parallel tool calls + subagent delegation
是工程信号，描述里能看出执行层和编辑器解耦的架构（OpenCode server 跑
在外面，VS Code 只是一个壳）。MIT/开源、装机命令是 `kilo.ai/install`
单行脚本，部署路径成熟。573 票 / 123 评是今日唯一上量级讨论的产品。

**信号价值**：IDE 内 coding agent 的产品边界正从"对话式补全"转向
"agent 编排面板"——并行子任务、可视化 diff、模型 A/B 都开始进入主
界面。对做 dev tools / agent 编排的方向，Kilo 这种"开源 + 把别人内核
当依赖"是值得跟进的范式：不重造执行层，专攻调度和 UI。

### 2. Hestus · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/hestus)

**定位**：CAD 环境内的"原生 autocomplete"，不是 AI 聊天框、不是插件
弹窗，而是边画边预测下一步操作（"2.5x 更快、4x 更少点击"）。

**差异点**：CAD 是 AI 工具至今几乎没碰的领域——SolidWorks/Fusion
插件市场长期被参数化模板和宏占据。市面上的"AI for CAD"基本是
text-to-CAD（Spline AI、Zoo），Hestus 走的是 Copilot 那条路：理解当前
设计意图、预测下一步，不要求用户改习惯。这是 CAD 类工具第一次
真正复用"IDE 内嵌 agent"范式。

**技术判断**：要做到"native autocomplete"必然要 hook 进 CAD 的
geometry kernel 或 UI 事件流，光靠 LLM prompt 不够，需要本地几何
状态建模 + 实时推理。Pitch NYC 标签，团队应当还在早期，但工程门槛
比一般 SaaS 高一截。

**信号价值**：Copilot 范式正从代码外溢到专业垂直工具——CAD、CAM、
PCB、3D 建模都还有 Cursor-shaped hole。判断方向时可以盯"哪些专业
软件工作流是高频小操作 + 强意图可预测"，那里就是下一个 autocomplete
机会窗口。

### 3. Intuned Agent · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/intuned)

**定位**：用 AI agent 写并维护生产级 browser automation——你描述要什么
scraper / RPA 流程，它产出 Playwright 代码、在真实站点上验证，部署
到 Intuned 跑，并在网站结构变化时自动修复。

**差异点**：和 Browserbase / Browser Use 这类"用 LLM 直接驱动浏览器"
的方案不同，Intuned 输出的是真实 Playwright 代码，跑在受控环境里。
这等于把 agent 当"代码生成 + 维护工"用，把可观测性和稳定性留给传统
RPA 栈。对企业场景这是更合理的分工：LLM 不在 critical path 上每次都
推理，而是在"代码漂移"时介入修复。

**技术判断**：核心难点是 selector 稳定性 + 自动修复链路（截图 diff /
DOM diff → 触发 LLM 重写 → 回归验证）。这套 pipeline 比生成代码本身
难。能稳定上生产的少数几家会很有价值。

**信号价值**：印证了 04-28 报告里我提到的"agent 不在每次执行里推理，
而是负责生成和维护"的分工正在成型。对做 agent infra 的方向，这是个
很清晰的信号：把 agent 推到 build/maintain 层，run 层用确定性代码，
能解很多企业落地问题。

PH 描述不够判断信号是否真的稳定到能做实时情绪分类——但若做到，
是 wearable 维度的真创新，不是套壳。

**信号价值**：CLAUDE.md 的关注列表里"novel I/O modalities"今日唯一
真正的代表。如果未来 agent 要拿到"用户当前认知状态"作为上下文，
这条路上的硬件就是 enabler。值得长期挂着。

**信号价值**：Computer Use 类产品需要一个开源 baseline，Openclick
是少数能填这个空位的早期项目。对自建 agent 工具链的方向，这是
可以直接 fork 改的素材，不是要花订阅费的 SaaS。

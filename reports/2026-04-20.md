——在 VS Code 里是一组快捷键，在 Zoom 里是另一组（静音/切共享），在

**差异点**：Stream Deck 的可配置键位在 2014 年就有了，但它是 **用户手动

纯可编程键盘，Dune 把 "agent permission prompt" 写进了官宣用例。

**技术判断**：硬件不难，难在 macOS 上拿到前台应用 + AX 辅助功能信号的

Claude/Openclaw 集成，应该是走的 local socket 或类似 BLE 通道）。能跑出
99 条讨论说明真实使用场景被买账，不是玩具。

**信号价值**：agent 正在从 "屏幕内一个对话框" 外溢到 **外设 I/O 层**。
如果 agent 要长期在用户身边跑，approve/stop 这类控制信号就不适合留在
软件界面里——物理按钮是自然的归宿。做 agent 工具链的值得开始考虑
"**命令-批准分离**" 的人机工效：决策在屏幕里，关键动作在物理键上。

### 4. TorchTPU · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/google) · 103 votes / 5 comments · topics: open-source, developer-tools, artificial-intelligence

**定位**：Google 官方 PyTorch-native TPU 后端。原有 PyTorch 代码几乎
零改动就能跑到 TPU 上，**Fused Eager 模式号称 50-100%+ 加速**，不再要求
把模型编成静态图就能起跑，能一路扩到 10 万+ 芯片集群。

**差异点**：之前 Google 推 JAX/XLA 是"重写一遍"路线，PyTorch/XLA 的 TPU
支持也是 lazy tensor + trace + compile 的慢启动体验。TorchTPU 把路径换成
"eager first, compile optional"——这正是 PyTorch 用户日常习惯，大幅降低
从 GPU 迁到 TPU 的冷启动成本。对抗的是 NVIDIA CUDA 的**锁定**，打的是
中大型训练负载。

**技术判断**：Fused Eager 说明他们在 op-level 把多个 kernel 在 eager 路径
就合并掉了——这是真正吃力的编译器工作，不是"包一层 API"。发布方是
Google 本尊，engineering 深度无需质疑。

**信号价值**：2026 年 **训练侧的硬件供应链竞争** 进入新一轮：推理有
Groq/Cerebras/SambaNova 分食，训练长期被 NVIDIA 锁死。TorchTPU 是
Google 拿 TPU 公有云正面回应——如果真能在 PyTorch 生态里 free-seat
起来，对 "买不到 H100 的团队" 是另一条逃生道。做 AI infra 的值得跑
benchmark 验证 tagline 中的 "50-100%" 到底在什么 workload 上成立。

### 5. GalaxyBrain · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/galaxybrain-com) · 93 votes / 2 comments · topics: productivity

**定位**：**local-first 的"知识即编程环境"**。页面里可以写变量、公式、
跨页引用，一页改值另一页自动同步。底层是机器上的一堆结构化 JSON 文件，
**自带 HTTP API + MCP server**，Claude Code / Codex / 本地模型都可以
指向同一个目录读写。

**差异点**：Notion / Obsidian 主打云同步 + markdown；Logseq 是 local-first
但没有 "变量+公式" 的电子表格语义；Roam 有 reference 但不是 programmable。
GalaxyBrain 的独特组合是 "**document + spreadsheet + JSON 文件 + MCP
adapter**" 四合一，且没有账号系统——100% 本机。

**技术判断**：JSON 文件即真相源 + HTTP API + MCP = 有意为 agent 设计的
数据层。公式引擎 + 跨页引用是个非平凡的 reactive graph，不是接口拼贴。
MCP 不是加上去装饰，是 **设计动机**——作者把 agent 读写知识库当一级
场景。

**信号价值**：今天 Top 5 里最能说明 "**local-first + agent-first 是同一
件事**" 的产品。当个人数据留在本机 + MCP 暴露接口时，不同模型 / 不同
agent 都能在同一份真相上协作，这比云端 SaaS 的"各自孤岛"可扩展得多。
做个人 productivity + agent 工具的，MCP 已经不再是"可选集成"，它是
**产品形态的一部分**。

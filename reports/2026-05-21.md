description 没给细节。

### 2. Slideshot · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/slideshot) | [官网](https://slideshot.ai)

**定位**：让 AI agent 通过 MCP 驱动你的 web app 跑一遍预设 flow，
全程录屏 + 加 zoom / 鼠标动效 / 片头，吐出一段可发布的 demo 视频和
GIF。

**差异点**：现有 demo 工具（Loom、Tella、Arcade、Supademo）要么靠人录、
要么靠交互回放。Slideshot 直接把"操作"这件事交给 agent——agent 自己
读 MCP 描述、规划点击路径、再触发录制。这是把 browser-use / Computer
Use 类技术落到一个**有清晰 PMF 的场景**：开发者最烦的"录 demo"。

**技术判断**：核心应该是 MCP 客户端 + 浏览器自动化 + 后处理流水线
（zoom 跟随光标、生成片头）。MCP 是关键——说明它能复用产品方提供的
agent 接口，比纯 DOM 爬虫更可靠。能不能稳定跑通复杂 flow
（含登录、表单、异步加载）是技术门槛。

**信号价值**：MCP 不再只是 dev-tool 圈的术语，开始进入**面向终端
产品的工作流**。Slideshot 是一个明确信号：MCP server 多了之后，
会出现一批"用 MCP 当通用 driver"的内容生成产品（demo / 截图 /
教程 / 自动化测试视频）。这是建 MCP server 的产品方都该想清楚的
twist——你的 MCP 接口正在变成别人产品的 SDK。

**定位**：Google 出的本地桌面 app，用来并发跑多个 AI agent，支持

**差异点**：和 Claude Code、Cursor、Continue 那类编辑器内嵌 agent
不同，Antigravity 是**独立桌面 app**——明确把"agent 调度"从 IDE 里
抽出来，作为一类新的桌面应用。和 Tycoon AI（#1）类似都在做
agent-of-agents，区别是 Google 把它工程化进了自家生态（Firebase、
Android、AI Studio 三件套），定位是**开发者主战场**而不是"一人公司"。

**技术判断**：能跑 scheduled background tasks 说明有常驻进程
+ 持久化任务队列。原生集成 Firebase / Android 意味着它能直接拿
你的 cloud function / device 当 worker，跨进程跨设备的 agent
执行。这个架构如果做扎实，会是 IDE-内嵌方案吃不下的场景。

**信号价值**：两个看点：(a) **桌面 app 作为 agent 容器**回归——
和过去十年 SaaS 浪潮反向，是因为 agent 需要常驻、需要本地文件
/ 进程 / 网络访问。(b) Google 在 coding agent 这条线上明显在加码，
和 Cursor / Anthropic / OpenAI 形成多极。但只有 18 条评论，
和它声量不匹配——可能是定向用户，也可能是早期版本反响一般。

**差异点**：Superwhisper / Wispr Flow / MacWhisper 这一波已经把

对应一类对订阅疲劳的核心用户。功能层面（press-to-talk hotkey、
12 语言、视频转写、可配后处理规则）是平推竞品的合理实现。

**技术判断**：Whisper on Apple Silicon 是成熟方案，关键工程在
延迟（按键到输出）、模型尺寸权衡、热键拦截不冲突。"Rainbow Mode"
这种细节说明产品方有审美在打磨体验，是好信号。

**信号价值**：和 Novi Notes 1.1（榜上第 14，local AI memory layer
for Mac）一起，今日是 **local-first AI 工具的一个小集中**。趋势
判断：随着 Apple Silicon 性能爬上来 + 用户对云端 AI 的隐私 / 成本 /
延迟越来越敏感，**本地推理 + 一次性付费**是除"按 token 计费 SaaS"
之外被验证可行的第二条路径。值得关注的是这类工具的分发渠道——
Setapp、独立站、PH，几乎不在主流 AI 工具榜单里被讨论。

**差异点**：现有 robotics sim 平台（Isaac Sim / MuJoCo / Genesis）

**技术判断**：从描述看是 procedural generation + randomization
pipeline，可能基于 USD / Isaac 资产格式。挑战在如何让生成的
3D 在物理引擎里跑出"真实"的接触动力学。如果做扎实，是
robotics foundation model 训练数据这一层的关键基础设施。

**信号价值**：26 评论 / 58 票 = **0.45 的极高讨论比**，说明社区
有明确兴趣。机器人领域真正的卡点正在从"算法"往"数据 pipeline"
转移——这和 LLM 三年前的演进路径很像。看好这一层（sim 数据
合成、real-world data 收集）出 1-2 家头部基础设施公司。
对做 agent 的人启示：**把 sim 当 generator 用**这个范式（生成
合成训练数据 vs. 找真实数据）值得迁移到非机器人场景。

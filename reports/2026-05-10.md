## 🔥 Top 5

### 1. deepsec · ⭐⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/vercel) | [Website](https://www.producthunt.com/r/T5UPS2CJT3MEUS)

**定位**：Vercel 开源的 AI 编码 agent 安全护栏，跑在你自己的基础设施上、用你自己的 key、
扫你自己的代码——而不是再做一个 SaaS 中间层。

**差异点**：现有 AI coding 安全方案（Snyk、Semgrep AI、GitHub Advanced Security）
要么是云端 SaaS，要么是事后扫描。deepsec 的定位更像是"给 Claude Code / Cursor 这类
agentic IDE 加一层本地 harness"，把秘钥、代码、推理过程都留在用户侧。和 Vercel 此前
的 v0 / AI SDK 战略一脉相承——他们在 AI 工具链每一层都不愿做云依赖的入口。

**技术判断**：从命名（harness）和"on your infrastructure, with your keys"判断，
这不是 wrapper，而是一个可嵌入到 CI / 本地 dev loop 的策略执行框架。需要看 repo
才能判断是规则引擎 + LLM 校验，还是更复杂的 sandboxing。但 Vercel 出品 + 开源
+ MIT/Apache 几乎是确定的——单凭这个信号就值得跟踪。

**信号价值**：印证了一个我们一直在押注的方向——**agent 安全将从云端 SaaS 退回到
本地 harness**。当 agent 直接接触代码和密钥时，把信任边界放在远端服务器上是不可
接受的。Vercel 选这个角度切入，等于给整个行业打了一个 reference。如果你在做
developer-tools 或 AI agent infra，这是必读案例。

### 2. ClawSecure · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/clawsecure) | [Website](https://www.producthunt.com/r/CBB34LD6UL5TT2)

**定位**："给 AI agent 做杀毒"——预安装扫描 + 运行时监控 + 内嵌 Security
Companion Agent + sub-200ms 的 Verification API。覆盖 OWASP ASI 全 10 项。

**差异点**：和 deepsec 形成了有意思的对照——deepsec 关心的是"你自己的 coding agent
不要写出问题代码"，ClawSecure 关心的是"你装的第三方 agent 不要本身就是恶意的"。
后者其实是一个新得多的市场——MCP 生态、agent marketplace、AI app store 这些都还
没有信誉体系。"41% of top agents are dangerous" 的数字虽然带营销味，但方向真实。

**技术判断**：sub-200ms 的延迟目标暗示这是个静态分析 + 行为指纹库的组合，配上一个
轻量 LLM 做语义判断。和传统杀毒的"特征码 + 启发式"几乎是 1:1 对应。OWASP ASI 是
今年才出来的标准（Agentic Security Initiative），紧跟标准本身就是个加分项。

**信号价值**：**"agent 杀毒"作为一个独立品类正在形成。** Antivirus 的隐喻很强——
意味着发行者期待用户像装 ESET 一样常驻在系统里。如果 MCP 服务器和 agent 包真的开始
通过分发渠道扩散（类似 npm 之于供应链攻击），这种产品会变成基础设施。

### 3. AgentPeek · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/agentpeek) | [Website](https://www.producthunt.com/r/I5VUUCMKZ6ZLDB)

**定位**：把 Claude Code 和 Codex 的运行状态塞进 Mac 刘海（notch）——sessions、
permissions、token 用量、本地 dev server 一目了然。Session 数据全部留本地。

**差异点**：现在的 agent observability 工具（Langfuse、Helicone）都是面向服务端
trace 的，假设你在跑生产 LLM 流量。AgentPeek 服务的是完全不同的人群：在 IDE 里
让 Claude Code 后台跑任务、希望知道它"在干嘛 / 烧了多少 token / 卡在哪个权限"
的开发者。Mac notch 作为入口很有意思——它在视觉上专属于"系统级的、永远在那里
的东西"。

**技术判断**：技术含量不算高（NSStatusBar / NSWindow 配合 Claude Code 的本地
日志/IPC），但**产品判断很准**。"Session data stays local" 这句话说明他们直接
读取 Claude Code 写在本地的 session 文件，没走任何中转 API。这个集成模式本身就
是一份关于 Claude Code 内部结构的廉价文档。

**信号价值**：两个交叠的信号——(a) 新的 I/O 表面（Mac notch、Touch Bar 类的
ambient UI）开始被用于 agent；(b) Claude Code 作为平台已经有了第三方观测工具，
说明它的本地状态格式正在事实标准化。如果你在做 dev-tools，值得看下他们是怎么读
session 的。

### 4. Keel · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/keel-3) | [Website](https://www.producthunt.com/r/MHKCZ5OJ47MKOE)

**定位**：本地优先的 AI 助手桌面应用（macOS/Win）。把对话里"重要的东西"抽取成
本地磁盘上的 markdown 文件——bring-your-own-model（Claude / GPT / OpenRouter
/ Ollama），开源 MIT。

**差异点**：和 Mem / Rewind / 主流 LLM 的"云端 memory"路线完全相反。最近一年
"memory" 这个词被 OpenAI / Anthropic / Notion 各家定义得很乱，Keel 选了一条
最简陋也最可审计的路线：**memory = 你磁盘上的纯 markdown**。这种"用户可以
diff 自己 AI memory"的范式，在隐私意识强、又懂技术的群体里有市场。

**技术判断**：技术栈大概率是 Electron/Tauri + 本地 SQLite 索引 + 直接调上游
模型 API。"plain markdown on disk" 不是技术挑战，而是产品决定。MIT 开源 +
BYOM 让它和 Obsidian 那类工具属于同一文化阵营。

**信号价值**：**"memory portability"** 正在变成 local-first AI 工具的核心卖点。
当 ChatGPT、Claude、Notion 都把 memory 锁在自家云时，"memory 在你磁盘上"成了
一个清晰的差异化点。如果你在做 personal AI / RAG，这是一个低成本可以学习的
对照样本。

### 5. LumiChats Offline(free) · ⭐⭐⭐
[PH](https://www.producthunt.com/products/lumichats-offline) | [Website](https://www.producthunt.com/r/D2CFOVK7DPCEEE)

**定位**：完全离线的桌面 AI 客户端，基于 GPT4All，支持 Mistral / LLaMA / Qwen /
DeepSeek 以及自家微调模型，LocalDocs 支持本地 PDF / 文档对话。Win / Linux / mac
全平台，无 GPU 也能跑。

**差异点**：和 LM Studio / Jan.ai / GPT4All 本身的功能高度重叠。亮点不在"能离线
跑"，而在"零数据收集 + 100% 免费 + 自带 LocalDocs"的明确组合。和 Ollama 那种
极客向 CLI 路线不同，它走的是"我妈也能装"的 GUI 路线。

**技术判断**：明确说"built on GPT4All"——本质上是 GPT4All 的发行版/UI 重打包，
工程量主要在 UX 和模型 catalog 维护。**这是个分发层产品，不是基础设施**。

**信号价值**：本地推理的"消费级化"在加速——三个月前这类产品的 tagline 还在强调
"private"、"local"，现在普遍强调"零数据收集"、"无需 GPU"、"免费"。说明目标用户
正在从极客往普通用户外扩。但同时也意味着这个赛道的差异化空间在收窄——再过一年，
能存活下来的可能就是和具体场景深度耦合的（如 Keel 这种）。

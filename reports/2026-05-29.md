干净抽象，值得在 agent infra 方向上注意。

**定位**：把任意 REST/GraphQL/SOAP/gRPC API 自动转成 MCP 工具，自带 typed schema、
auth、rate limit、响应处理，让 LLM 通过统一 MCP 接口调用企业 API。
**差异点**：和官方 `mcp-server-openapi`、Cloudflare 的 MCP gateway、Zapier MCP 比，
卖点在多协议覆盖（不止 REST，还包括 SOAP/gRPC——明显瞄准传统企业 IT 栈）和 enterprise-

**技术判断**：核心是 Schema introspection + 类型推导 + 调用代理。SOAP / gRPC 的
schema 描述比 OpenAPI 更结构化，工程量主要在四种协议的统一抽象层。如果他们真的把
auth flow（OAuth2、SAML、mTLS）封装好，价值不小。
**信号价值**：MCP 正在事实上成为 agent ↔ tool 的标准协议层（OpenAI、Anthropic、
Google、Cursor 都在跟进）。"任何 API → MCP" 是 unavoidable 的中间件赛道，谁能率先做
出企业级、协议齐全的方案谁就占据基础设施位。值得跟踪。

### 3. Integuru · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/integuru) | [r/BHYPI3BVUU7MB3](https://www.producthunt.com/r/BHYPI3BVUU7MB3)

**定位**：给那些没有官方 API 的平台"反向生成"一个真实 API——不用 browser 自动化、
不用 RPA，直接和后端协议对话，宣称 99.9% 成功率、3s 延迟。
**差异点**：和 Browserbase、Reworkd、Skyvern 这类基于 Playwright/CDP 的 web-agent
路线完全相反，走的是"嗅探 + 协议逆向"路线（很可能是 HAR 录制 → 请求模式提取 →
auth flow 模拟）。它在赌的是：未来 agent 的主要瓶颈是 reliability，而 browser
automation 在这一点上是死胡同。
**技术判断**：技术上能不能成立是个 open question——很多企业 API 后端有反爬、
device fingerprint、动态 token，纯 backend 协议模拟非常脆弱。但他们承诺
"auto-healing + 24/7 on-call maintenance"，说明本质上是 API-as-a-service
（有人值班维护底层逆向工程），不是纯产品。
**信号价值**：反映了 agent 工具链上的一个分歧：browser-based vs. protocol-based。
browser 路线 demo 漂亮但生产环境脆弱，protocol 路线对早期更可行但维护成本高。
做 agent infra 的人需要明白这个 trade-off，而且这类"逆向 API 即服务"在 SaaS
integration 长尾市场的需求一直存在。

### 4. GPS · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/gps-2) | [r/VWVFWSTQX5JNSD](https://www.producthunt.com/r/VWVFWSTQX5JNSD)

**定位**：给 coding agent（Claude Code、Cursor、Codex）做的本地记忆层——记住 repo
约定、过往决策、教训，在编辑前主动注入相关上下文，避免反复犯同样的错。
**差异点**：和把规则全部塞进 `CLAUDE.md` / `.cursorrules` 的做法相比，GPS 走的是
"细粒度、按符号锚定"——记忆挂在具体函数/文件上，不是一个膨胀的全局 prompt。和
Mem0、Letta 这些通用 agent memory 框架比，它专门做 coding agent 这个垂直场景，
所以能利用 AST / 符号表来定位上下文。
**技术判断**：技术栈应该是 tree-sitter 解析 + 符号索引 + 向量检索 + MCP 接口。
难点在"什么时候 surface 什么记忆"——recall vs. precision，错误的注入会污染上下文。
本地运行 + CLI-first 的定位是对的，避免了把代码上传到云端。
**信号价值**：context engineering 正在被验证是 coding agent 性能的关键瓶颈，比
模型本身更重要。这一类 "agent memory 中间件" 会出现一波——和我自己做 agent 的
方向高度相关：CLAUDE.md 这种全局 prompt 注定不可扩展，按需检索 + 符号锚定才是
正解。

[PH](https://www.producthunt.com/products/claude-code) · 196 votes / 9 comments · topics: software-engineering, developer-tools, artificial-intelligence

**定位**：Anthropic 给 Claude Code 加的 slash command，把当前分支或 PR
扔到云端 sandbox，启动一组并行的 reviewer agent，每个声称的 bug 由独立
agent 复现验证后再上报。Pro/Max 用户可用。

**差异点**：和 GitHub Copilot Review、CodeRabbit、Greptile 这类"AI 阅读
diff 给评论"完全是两条路线——它是把 review 当成 **agent 任务**：每个
reviewer 在 sandbox 里能跑代码、能编译、能验证假设。"独立验证后再报告"
是关键差异，这是为了压住 LLM review 最大的痛点（hallucination + false
positive 噪声）。

**技术判断**：从描述看，架构上是 fan-out reviewer 集群 + per-finding
verification step，本质是 hierarchical multi-agent。落地难点不在 prompt
而在 sandbox 的可扩展性和成本控制——并行 agent + 复现验证意味着一次
review 可能烧掉数十次完整 sandbox 启动。Anthropic 自己出，证明它跑得起
这个成本。

**信号价值**：值得记录的趋势是"agent 自我验证"开始成为产品级功能——
不再依赖 LLM 单次输出的可信度，而是用执行结果去过滤 LLM 输出。对做
agent 框架的启示：reviewer/critic 这一层可能比 generator 更值得投资，
"先生成，后验证"会逐渐变成默认架构。另外注意：Anthropic 自己往上叠
slash command 而不是开放 API，说明 Claude Code 正在从工具向平台演进。

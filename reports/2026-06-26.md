### 1. Agent Arena · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/agent-arena) · 346 votes · 65 comments

**定位**：给自主 agent 办一个公开"擂台"——任何人可以创建/参加比赛，agent 在真实任务上竞争、积累 reputation、累计奖励，并基于结果迭代。本质是把单点的"benchmark 评测"做成一个持续运行、有经济激励的社交化网络。

**差异点**：现在 agent 评测要么是静态 benchmark（SWE-bench、AgentBench、GAIA），要么是公司内部的私有 leaderboard（HumanEval、内部 eval set）。Agent Arena 走的是"竞技场 + 声誉系统 + 奖金"路线，更接近 Kaggle / Topcoder 的形态搬到 agent 上。和 lmsys / Chatbot Arena 也不一样：那个评的是模型回答，这个评的是 agent 端到端完成任务。

**技术判断**：背后是 NetMind.AI（Kai Zou 是 founder + CEO，PH 上账号挂的就是 NetMind）。NetMind 之前在做去中心化 GPU 调度和模型推理网络，做一个 agent 竞技场算是顺势把客户拉到自家算力里。难点不在评分系统，而在"如何让结果可信"——agent 跑在哪台机器、是不是用了人工干预、是不是偷看了其他参赛者的轨迹。如果他们能把这套 verification 做扎实，是有真壁垒的；如果只是社区投票，就和现在的 GitHub awesome list 差不多。

**信号价值**：65 条评论 / 0.188 评论率是今日榜单最讨论度的产品之一，说明 agent builder 圈子真的有这个需求——大家训出来一个 agent 之后没地方比较。这是 agent 生态成熟到一定阶段必然出现的形态（类似 Kaggle 之于 ML）。值得跟踪：评分机制能否做得让人服气，是这类产品的生死线。

### 3. note.md · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/note-md) · 278 votes · 45 comments

**定位**：Mac 原生的 local-first 研究工作台。读论文、管参考、写 markdown 笔记、引用证据，全部在本地完成；上层接 local LLM 作为研究记忆。目标是把 Zotero + Obsidian + PDF reader + 写作工具的拼图缝成一个产品。

**差异点**：和 Zotero（管参考、不能写）、Obsidian（写得好、但论文流程要插件拼）、Reflect/Heptabase（写得好、但不是 research-first）、ChatGPT Projects（云端、隐私不可控）都有重叠，但定位是"local LLM 作为论文阅读和记忆的一等公民"。和 NotebookLM 的根本区别在 local-first——研究材料不离开机器。

**技术判断**：tagline 里"now a local LLM Memory"说明他们走的是 RAG 本地化路线，论文/笔记切片后塞进本地向量库，由本地模型（大概率 Ollama / llama.cpp / MLX）做检索增强。这条路在 M 系列 Mac 上是可行的，但有几个工程坑：(a) 多模态 PDF 提取的质量（公式、图表）；(b) 本地向量库的更新策略；(c) 切换模型/embedding 后历史 index 是否兼容。45 条评论的讨论密度说明研究人群确实在用，不是营销刷的。

**信号价值**：精确命中 CLAUDE.md 关注的"local-first / on-device inference / privacy-preserving"。研究人员是 local LLM 第一批真正掏钱的用户群——他们的数据敏感、对延迟和上下文长度的要求又高。每一次有团队认真做 local-first 知识工作台都值得记下来，等几年后这个赛道成熟会有人胜出。

### 4. LockIn MCP · ⭐⭐⭐
[PH](https://www.producthunt.com/products/lockin-mcp) · 116 votes · 21 comments

**定位**：把"网站屏蔽 / 专注模式"做成一个 MCP server。不再装 Chrome 插件，而是告诉你的 agent（Claude、Cursor 等）："锁我两个小时"，agent 通过 MCP 调用去操作 OS / hosts 文件做封禁，并且——和插件不一样——agent 自己也无法轻易绕过。

**差异点**：传统反分心工具（Cold Turkey、Freedom、Opal、SelfControl）都是用户主动设置定时。LockIn MCP 把这件事变成"agent 的能力"：你的 coding agent 在帮你写代码时可以主动决定"你今天上下文已经被切碎太多次，我要帮你封 30 分钟 Twitter"。这是一个非常细的产品 idea，但它代表了一个新形态——**人和 agent 协商对自己的控制权**。

**技术判断**：MCP 协议本身实现量不大，难点在 OS 级别强制：macOS 需要 admin 权限改 hosts 或者用 ScreenTime API，Windows 类似。21 条评论里大概率有人在问"agent 怎么解锁"——这是核心信任问题：如果 agent 触发也能触发解锁，就和插件一样可被绕过；如果只能通过物理操作解锁，又会触发 UX 灾难。

**信号价值**：MCP 作为应用形态正在向意想不到的方向扩展——不只是"暴露 SaaS 让 agent 调用"，而是"给 agent 一个新动作"。这条产品如果能跑通，会是一类新模板：把以前只能用户主动操作的 native 能力（屏蔽、休眠、锁屏、系统设置）暴露成 MCP tool，agent 就有了"作用于用户行为本身"的接口。比起 SaaS-as-MCP，这种"native capability as MCP"的方向被低估。

### 5. DMV by Agent Community · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/dmv-department-of-machine-verification) · 111 votes · 10 comments

**定位**：一个面向 agent 的命名/身份注册局。在向 ICANN 申请 `.agent` 顶级域名（TLD），社区先开放预注册，每个 agent 拿到一个"身份卡"和一个 `<name>.agent` 名字。目标是给 agentic web 提供一个不被任何单一公司控制的命名层。

**差异点**：现在 agent 的"身份"基本是 OAuth token 或者厂商私有 ID（OpenAI 的 assistant id、Anthropic 的 agent profile、各家 MCP 服务器自定义的 auth header）。`.agent` TLD 想做的事和 DNS / DID（去中心化身份）想做的事在精神上类似，但 surface 是大家最熟悉的"域名"形态。如果真的被 ICANN 批了，会是 agent 互操作的一个底层 primitive。

**技术判断**：这是一个"基础设施 + 政治"的混合工程。ICANN 审批一个新 gTLD 是漫长且贵的（六位数美元起步、几年审批周期），描述里"29,000+ members and 7,000+ companies"是为审批做的 community-support 证据。技术侧没什么难点，难的是制度建设：解析谁来跑、争议怎么解决、保留词怎么定。Agent Community 看起来是踩在了一个非常对的位置上抢窗口期。

**信号价值**：这是今天榜单里最长期的赌注。短期看像"另一个 web3 命名服务"，但和 ENS / Handshake 不同的是——他们走的是 ICANN 官方通道，不是去做替代根域。如果赌赢了，所有未来的 agent 都可能挂一个 `<x>.agent` 身份；如果赌输了，团队还是吃到了一波"agent identity"叙事的关注。值得密切跟踪 ICANN 审批进度（公开信息可查）。

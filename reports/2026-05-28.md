### 1. Memori · ⭐⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/memori-labs) · 151v / 25c · topics: open-source, developer-tools, ai

**定位**：给 agent 用的"行为记忆"基础设施。区别于市面上几乎所有"对话记忆"
方案（mem0 / Letta / Zep 等都是从 user↔assistant 对话历史里抽事实），Memori
直接从 agent trace（工具调用、参数、返回值、决策路径、workflow 步骤、最终
outcome）里结构化产出长期记忆，让 agent 记住"我上次怎么做这件事"而不是"用户
上次说了什么"。

**差异点**：
- 对话记忆 vs 执行记忆是两个层。对话记忆能记住"用户偏好咖啡不要糖"，但记不住
  "上次连 Postgres 改用了 SSH tunnel 这个 trick"；trace 记忆专门解后者。
- 跑分：LoCoMo 81.95% accuracy，每 query 仅 1294 tokens（声称是 full-context
  的 ~5%）。如果数据可复现，意味着 95% inference 成本节省，对长时间运行的
  agent 来说是直接的经济性问题。
- 已经 15K GitHub stars / 200K+ downloads——不是 launch-day 才出现的，是
  PH 来背书一个已经被开发者验证过的库。

**技术判断**：agent-native memory 这个赛道之前一直被 conversation memory 一家
独大（mem0 26K stars），Memori 把语义层从 utterance 改到 trace，是真正
的"换了一个抽象"。能跑出 1.3K tokens/query 这个数字，背后大概率是
embedding + trajectory clustering + tool-result schema 化的组合，不是 RAG
+ summary 那一套。

**信号价值**：CLAUDE.md 关注的"AI agents 与多智能体编排"方向上最有
工程深度的一个。Memory infra 是 agent 经济性的核心瓶颈之一（Anthropic 自己
也在做 memory tool），第三方库能在这条线上跑出独立赛道，意味着
**agent 工具链开始分层**：编排层（LangGraph / Autogen）/ 执行层（OpenHands /
Aider）/ 记忆层（mem0 / Memori）/ 观测层（Helicone / LangSmith）。
对自研 agent 的方向是直接利好——可以挑层用。

  TTS 把 "(415) 555-..." 念成"四千一百五十五"折磨过。
- 词级时间戳 + IPA 支持：直接面向需要 lip sync、字幕对齐、二次后期处理
  的开发者。
- LiveKit / Pipecat / Vapi 三个 voice agent 框架的 adapter 都做了——直接对
  接 voice agent stack。
- on-prem 可部署：医疗、金融、政府这些 ElevenLabs 进不去的场景。

**技术判断**：sub-60ms + 自托管 + voice cloning 同时拿到，模型不会太大
（4B 以下估算），大概率是 streaming neural codec + flow matching / diffusion
decoder 的方案。grammar-aware normalization 不是模型能力，是前置 NLU pipeline
——这种愿意做"脏活"的团队工程素质通常靠谱。

**信号价值**：voice agent stack 在 2026 上半年开始分化——
**Composio / Vapi / LiveKit 抢编排层，TTS / STT 这一层在出现新的玩家**。
ElevenLabs 是 SaaS-only，本地化部署的 gap 是真存在的。这个产品直击 CLAUDE.md
的"local-first / on-device inference"和"voice modality"两条优先线。

### 2. Cosine Swarm · ⭐⭐⭐⭐

**差异点**：Cursor / Windsurf / Claude Code 目前都是 "一个 agent + 你" 的线性协作模式。Cosine Swarm 把多 agent 并行执行做成第一公民，用 PR 作为 human review 汇合点——"swarm 去探索，人在 PR 层做决策"。对长 horizon 的 refactor / migration 是更自然的形态。

**技术判断**：产品描述和 K2.6 的叙事呼应（见 Top 1），说明这不是巧合：**"subagent orchestration" 正在从工具模式变成产品模式**。单 runtime 跨 CLI/Desktop/Cloud 是务实的工程选择——说明他们在做 session 状态持久化和任务树共享，不是套壳。

**信号价值**：04-16 的 "Agent infrastructure" 趋势（Claude Code parallel sessions、Gemini CLI subagents）今天升级到**产品化 multi-agent orchestrator**。这一波要认真看 UX 和语义——谁能把 "多个 agent 跑崩了怎么办 / human review 放在哪" 想清楚，谁就拿到 SWE agent 的下一个身位。

### 3. Spectrum · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/spectrum-5) · 158 votes / 17 comments · topics: messaging, open-source, github, imessage-apps

**定位**：开源框架，把 agent 接到 iMessage / Telegram / WhatsApp 等现有消息平台，统一 API，agent 发一条消息由 Spectrum 处理格式转换和投递，声称 <1 秒。

**差异点**：slogan 一针见血——"Nobody is going to download an app or visit a website to use your agent"。之前这个分发层要么自己写 connector（苦力活，每个平台都有 rate-limit / 富文本差异），要么用 Make/Zapier（慢、贵、不适合 agent 实时输出）。Spectrum 把它做成基础设施。

**技术判断**：难点在 iMessage（非官方 API）和 WhatsApp（Business API 审批）。能宣称 <1 秒投递 + 统一 API，背后要么有自建桥，要么接了 iMessage Server 这类 gray-area 服务——值得去 GitHub 看看具体怎么实现。如果是前者，这是工程深度；如果是后者，合规有坑。

**信号价值**：**Agent distribution surface** 是 2026 年被低估的一层。模型和 agent framework 竞争到头，谁拿到 "用户原地的界面"（iMessage / WhatsApp / Slack），谁就绕过了应用商店和注册流。对做消费级 agent 的尤其关键。

### 5. YourMemory · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/yourmemory) · 88 votes / 7 comments · topics: open-source, storage, artificial-intelligence, github

**定位**：MCP-native 的 agent 记忆层，用 Ebbinghaus 遗忘曲线自动剪枝过期 context，声称减 84% token 浪费，LoCoMo benchmark 52% recall，100% 本地。

**差异点**：目前的 agent memory 方案是两极——mem0 / Letta 这类云托管，或者自己写 vector DB。YourMemory 走 "本地 MCP server + 带算法的剪枝" 路线。Ebbinghaus 曲线作为剪枝策略比单纯 LRU / TTL 有理论支撑（越久远的越倾向删，但偶尔复用的会被"加固"），是一个具体的 memory-policy 创新。

**技术判断**：自带 v1.3.0 Graph Engine（"Finds what you forgot to ask for"）说明除了 vector 还有实体/关系图检索。LoCoMo 52% recall 是一个具体可验证的数字（公开 benchmark），这点比大多数 "AI 记忆" 产品扎实太多——大部分友商连 benchmark 都不挂。

**信号价值**：**MCP 生态的 memory primitive 正在开始分化**——之前大家把 memory 当黑盒服务，现在开始拼算法（forget curve）和可控性（100% 本地 + 开源）。对自建 agent 框架的人来说，这比 "ChatGPT Memory" 那种闭源托管有工程价值。值得放进 evaluation shortlist。

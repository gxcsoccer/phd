### 1. ZooData · ⭐⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/zoodata) · 611 votes / 81 comments

**定位**：把任意 URL 变成"agent 直接可用的结构化 JSON"的抓取层。宣称
比原始 HTML/markdown 少用 ~75% token，附带 Amazon / TikTok 电商竞品维度
的预分析数据。API + CLI + MCP server 三件套。

**差异点**：市面上 Firecrawl / Apify / Browse.ai 都在做类似的事，但
ZooData 明确以"给 agent 消费"为设计原点——按字段计费而不是按调用计费，
输出直接 schema 化。MCP server 内置意味着 Claude / Cursor 里可以直接
挂载，不用自己写 tool wrapper。

**技术判断**：真做了工程。按字段计费+schema 输出说明底层不是简单 LLM
拉一遍 HTML，更像 selector 预训练 + LLM 结构化的混合管线。Amazon /
TikTok 的预分析层需要长期维护 scraper 和反反爬。不是套壳。

**信号价值**：这是本周第二个"agent 数据层"角度的产品（上周有类似定位
的产品）。**"抓取即服务"正在从 dev tool 迁移到 agent tool**——付费模型
（按字段 vs 按 API 调用）、协议接入（MCP first vs REST first）都在为
agent 用户重新设计。做 agent 项目做 web 数据摄入时值得对比一下现有方案。

### 2. Clark · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/clark-agent) · 484 votes / 62 comments

**定位**："自带云计算机"的 AI 同事——每个 Clark 实例都有独立的
browser / terminal / files / code，"关掉标签页，任务完成时再回来"。
支持分派给平行 specialist、定时运行、返回带证据的 artifact；提供
Clark Code 仓库内工作模式和 OpenAI 兼容 API。

**差异点**：定位介于 Devin（自主 dev agent）和 ChatGPT operator（云端
浏览器代理）之间，但把"每 agent 一台专属云机器"作为一等公民，且开放
兼容 API 让人可以像调用 GPT 一样把 agent 嵌入自己应用。相较 Anthropic
Computer Use（在你自己机器上跑）主打完全离开用户环境。

**技术判断**：能同时开出 browser + terminal + code + files + parallel
fanout + schedule + OpenAI-compat API，说明背后是一整套 sandbox 编排 +
持久化会话系统，不是简单 LLM + 单容器。工程量非常大；能不能稳定跑起来
需要看用户实测。

**信号价值**：**"agent-as-a-service" 商业模式正在成型**——不再卖模型
或 IDE 插件，卖的是"一个能干活的员工"。定价 / 计费单位很可能是
task-hour 或 artifact-count，而不是 token。做 dev agent 方向值得关注
它的 API surface 和 pricing 模型如何演化。

### 3. OpenMarkdown · ⭐⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/openmarkdown) · 206 votes / 59 comments

**定位**：本地优先的 markdown 编辑器，**你和你的 agent 同时编辑同一个
文件**。通过 CLI、agent 插件和 MCP 让 agent 读/写你正在编辑的文件；
无账号、无遥测、文件不离开本地。

**差异点**：市面上 Obsidian / Typora / iA Writer 都是给"人"用的；
Notion / Notion AI 则是把 AI 塞进服务器。OpenMarkdown 把"人 + agent
协同编辑本地文件"作为核心场景。MCP 内置意味着 Claude Code / Cursor
可以直接和它做双向读写，不用中间同步层。

**技术判断**：CRDT 或类似的冲突解决层几乎是必须的（不然人和 agent
并发写同一段会互相覆盖）。Local-first + MCP + editor 的组合在架构上
是有讲究的，能做出来说明团队理解 agent 协作的边界。

**信号价值**：**本地优先 + agent 协同**是本周命中 CLAUDE.md 关注领域
最精准的产品。相较"云端 AI 编辑器"（Cursor、Windsurf），它选择了
本地文件 + 明确边界的路线，隐私 / 供应链 / 离线能力都更好。对做本地
AI 工具的人来说是一个可复用的形式：不是 wrapper，而是把 agent 作为
"另一个编辑者"引入。

### 4. AgentGrid · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/agentgrid) · 69 votes / 58 comments · **ratio 0.84**

**定位**：把 AI coding agent、terminal、browser、notes、git 都放在一个
持久化的无限画布上。可以生成 role-based worker（builder / QA / reviewer
/ devops）+ orchestrator，跑"build → review → fix → validate"闭环。
本地运行，兼容 Claude Code 和 Codex，macOS / Linux / Windows 全平台。

**差异点**：多数 multi-agent 编排工具是 CLI 或纯 web dashboard，
AgentGrid 用"无限画布"隐喻让用户像看 Miro 一样查看 agent 状态和输出。
兼容 Claude Code 意味着不是自建 agent 引擎，而是复用现有 agent runtime
+ 提供可视化编排层。

**技术判断**：跨平台桌面 + 持久化 session 状态（agent / 面板 / git 都
恢复原位）+ role-based orchestrator 需要相当扎实的 IPC + 状态机设计。
"local, works with Claude Code / Codex" 明确不做闭环模型，只做编排层，
定位清晰。

**信号价值**：讨论 / 投票比 0.84 极高（69 票 58 评论），说明主题触到
了 hn/reddit 类做 dev agent 的用户群。**"给 agent 一个可视化的窗口"
是本周继续升温的方向**——上周有类似"看板+agent"的产品出现，这次进一步
落到桌面编排。

### 5. Postfleet · ⭐⭐⭐⭐
[PH](https://www.producthunt.com/products/postfleet) · 21 votes / 19 comments · **ratio 0.90**

**定位**：给 AI agent 的"防注入邮件基础设施"。所有入站邮件在 agent 读
到之前都会被验证签名、扫毒扫垃圾、**扫 prompt injection**，然后以
已解析 / 已理解的形式送到 agent。出站幂等，可加人工审批闸门。提供 6 个
MCP tool 和 REST API。

**差异点**：多数"AI + email"产品在做的是 draft 生成或分类，Postfleet 反
着做——它假设 **inbox 是 agent 的攻击面**，产品的核心是过滤而不是生成。
市面上 prompt injection 防御工具多是 SDK / firewall（LlamaFirewall、
Rebuff 等），Postfleet 把它包成一个特定 channel（邮件）的托管服务。

**技术判断**：Prompt injection 检测本身是活跃研究领域（Vigil、
PromptGuard、Anthropic 的分类器）。做成邮件 gateway 需要把 MIME parse
+ 附件沙箱 + 语义扫描 + 幂等发送整合起来，工程复杂度不低。产品收敛在
"专门给 agent 用"，避免了和 Superhuman / Missive 等消费级邮件正面撞车。

**信号价值**：这是**"agent 安全基础设施"**开始细分到具体 channel 的
信号——不再只有通用 firewall，而是"给某个 channel 的 agent 用的安全
gateway"。做 agent 项目、agent 消费 tool 数据源时值得预判：数据源
和外部通道都需要一层"agent-aware sanitize"，早晚会出现更多类似产品
（voice / SMS / Slack channel 版本）。

### 5. Assemble · ⭐⭐⭐
[PH](https://www.producthunt.com/products/assemble-2) · 94 votes / 4 comments · topics: open-source, developer-tools, artificial-intelligence, github

**定位**：开源的 "AI 工作配置生成器"——一个 `/go` slash command +
memory + spec-driven workflow，**零运行时、跨 21 个 AI 平台**（Claude
Code / Cursor / Windsurf / Continue 等等，大概率全覆盖）。

**差异点**：现有路径要么是各厂自己的 `.claude/` / `.cursor/rules` /
AGENTS.md 各写一套，要么是 OpenCtx / continue config 这种 IDE 特定
方案。Assemble 的野心是**把 "给 agent 的上下文和指令" 标准化成一份可
移植配置**——类似 package.json 之于 Node。关键词是 "zero runtime"：
不是 MCP server 这种需要跑起来的东西，纯是配置编译产物。

**技术判断**：本质是 "编译器 + 模板"，难度主要在 21 个平台的 rule 语法
差异调研（Claude Code 的 skills/hooks、Cursor 的 rules、Windsurf 的
workflows 都不一样）。不是模型工程，是生态工程。开源 + 零运行时的组合
降低了用户试用成本，策略上聪明。

**信号价值**：agent 配置正在产生**跨生态抽象层需求**。单团队就在不同
工具间切换（CLI 里用 Claude Code，IDE 里用 Cursor，CI 里用自定义
script），都需要重写同一份 "项目规则 / 记忆"。谁能定义这份 lingua
franca，谁就握着下一代 .editorconfig 级别的标准。想清楚 "记忆/指令/
规则" 这三样东西的跨工具可移植性，是 2026 的一个重要问题。

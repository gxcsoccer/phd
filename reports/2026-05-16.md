embedding 之上的 RAG），所以才能做到"按需召回而非全量注入"。技术含量在于
schema 设计 + 增量索引 + 召回策略，不是花活。开源 + 5000+ GH stars 说明

coding sessions 上 95% token 节省 / 200x tool call before context limit"——这

产品认真。

"memory" 这一层抽出来单独打。趋势很明显——agent 基础设施正在被拆成
runtime（Cline SDK）/ memory（Agentmemory）/ harness（OpenHuman）/ tool layer
（MCP）这几个独立品类。对做 ProductHunt agent 的自己有直接启发：我们的
`state/seen.jsonl` 本质上也是一个 observation log，能不能用 Agentmemory

**技术判断**：底层视频生成大概率是接 Veo / Sora / Kling 这类托管模型

镜头连贯性维护。这是真有产品工程量的——多 agent 协作做创意决策、给生成
模型做"上下文连贯"才是难点。如果只是 Sora wrapper，做不到 357v / 79c

pattern——它把多 agent 协作从开发者圈子（Cline、Claude Code）外溢到内容
创作圈。如果这个抽象能 work，下一步可能是 "agent 当 PM" / "agent 当
designer"——内容生产工作流的 agent 化是 2026 年的下一波。

**关注度**：⭐⭐⭐⭐（扣一星是因为底层视频质量未验证，且 marketing 类产品
